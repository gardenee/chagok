## 제품 사상

차곡은 단순한 지출 기록 앱이 아니다.
두 사람의 경제적 대화가 일어나는 공간이다.
각자의 소비를 투명하게 공유하고, 함께 만들어가는 재정 습관을 지원한다.

**슬로건**

- "따로의 소비, 같이의 차곡"
- "우리 둘의 내일이 차곡차곡 쌓여요"

## 디자인 원칙

### 클레이모피즘 (Claymorphism)

찰흙을 빚어놓은 듯한 귀여운 입체감과 따뜻한 색감.
딱딱한 금융 앱이 아닌, 일상에 녹아드는 따뜻한 도구.

**규칙**

- 모든 카드/버튼: `rounded-3xl` (마시멜로 느낌)
- 그림자: 메인 컬러를 연하게 섞은 소프트 섀도우 (딱딱한 검정 그림자 금지)
- 아이콘: Lucide, 굵기 두껍게, 이모지 사용 금지 아이콘 사용
- 색: 컬러 팔레트 외 임의 색상 사용 금지

### 애니메이션

- 지출 입력 시: 리스트 아래에서 위로 "차곡!" 쌓이는 모션 (Moti)

## 개발 지침

### MVP 1 범위 (현재 집중)

아래 기능만 구현한다. 범위 밖은 "MVP 이후"로 미룬다.

**포함**

- 소셜 로그인 (카카오, 구글)
- 커플 연동 (초대코드 + 링크)
- 가계부 이름 설정
- 지출/수입 CRUD
- 나/파트너/함께 태그
- 카테고리 커스텀 (이름/이모지/컬러/예산/순서)
- 홈 대시보드 (카테고리별 예산 현황)
- 캘린더 (지출 연동 + 일정 추가)
- 댓글 (실시간)
- 고정지출 등록
- 실시간 동기화
- 푸시 알림 (파트너 지출, 댓글, 고정지출 리마인더)

**MVP 이후 (지금 건드리지 않음)**

- 리액션 스티커
- 지출 분할 계산
- 자산 관리
- 통계 리포트
- Android 빌드
- Web 지원

### 코드 원칙

- **단순하게**: 과도한 추상화 금지. 지금 필요한 것만 만든다.
- **NativeWind 우선**: `StyleSheet.create` 금지. 모든 스타일은 `className`으로. 예외적으로 NativeWind가 지원하지 않는 속성(textShadow 등)만 인라인 스타일 허용
- **폰트**: `StyleSheet`에 `fontFamily` 하드코딩 금지. `tailwind.config.js`에 등록된 `font-ibm-bold` / `font-ibm-semibold` / `font-ibm-regular` 클래스 사용
- **컬러**: `tailwind.config.js`에 등록된 팔레트(butter, brown, cream, peach, lavender) 외 임의 색상 className 사용 금지
- **TypeScript 엄격하게**: `any` 금지, 타입 명시
- **컴포넌트 분리 기준**: 같은 UI가 2곳 이상 쓰일 때만 분리
- **상태관리**: 서버 상태는 React Query, 클라이언트 전역 상태는 Zustand, 로컬 상태는 useState
- **에러처리**: 사용자에게 보이는 에러는 반드시 한국어로

### 네이밍

- 파일: `kebab-case.tsx`
- 컴포넌트: `PascalCase`
- 함수/변수: `camelCase`
- 상수: `UPPER_SNAKE_CASE`
- Zustand store: `use[Name]Store`

### 커밋 규칙

```
feat: 새 기능
fix: 버그 수정
style: UI/스타일 변경
refactor: 리팩토링 (기능 변화 없음)
chore: 설정, 패키지 등
```
